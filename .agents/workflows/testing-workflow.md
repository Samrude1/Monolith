# Testing and QA Workflow

This workflow ensures that game logic, state machines, and utilities generated by AI or written by developers are verified through automated tests, maintaining reliability and preventing gameplay regressions.

1. **Architect Test Plan**:
   - Before writing tests, review `architecture.md` and `game-design-context.md` to understand the mechanic's intended behavior.
   - Identify the critical paths, edge cases (e.g., passing through walls at high speeds), and boundary conditions.

2. **Develop Tests** (`/test`):
   - Write automated unit or integration tests for the implemented mechanics.
   - Focus tests on pure logic: math utilities, collision detection algorithms, state machine transitions, and inventory management. Avoid writing tests that require complex rendering contexts unless specifically set up for E2E.
   - Ensure tests are deterministic.

3. **Execute & Verify**:
   - Run the test suite to ensure all new and existing tests pass.
   - If tests fail, diagnose whether the bug is in the game logic or the test setup. Fix accordingly.

4. **Review**:
   - Verify that test coverage is adequate for the new mechanic.
   - Ensure test descriptions clearly explain the *gameplay rule* being verified.
